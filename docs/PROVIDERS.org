* External LLM Provider Guide

Overview of all external LLM providers supported by Ollama Buddy.  This covers cloud APIs, local inference servers, pricing, free tiers, and recommendations.

The same =ollama-buddy-provider-create= system works for both remote cloud APIs and local OpenAI-compatible servers (LM Studio, llama.cpp, vLLM, etc.).

** How Provider Registration Works

Ollama Buddy uses a generic provider system built on =ollama-buddy-provider-create=.  There are three ways to add a provider:

*** Dedicated module (existing providers)

These have their own =.el= file with =defcustom= variables:

#+begin_src elisp
(require 'ollama-buddy-openai)
(setq ollama-buddy-openai-api-key
      (auth-source-pick-first-password :host "ollama-buddy-openai" :user "apikey"))
#+end_src

*** Direct registration -- cloud API

Add any cloud provider with a single call in your init file -- no extra files needed:

#+begin_src elisp
(ollama-buddy-provider-create
 :name "Groq"
 :prefix "q:"
 :api-key (lambda () (auth-source-pick-first-password :host "ollama-buddy-groq" :user "apikey"))
 :endpoint "https://api.groq.com/openai/v1/chat/completions"
 :models '("llama-3.3-70b-versatile" "gemma2-9b-it")
 :default-model "llama-3.3-70b-versatile")
#+end_src

*** Direct registration -- local server

Point at any local OpenAI-compatible server the same way (no API key needed):

#+begin_src elisp
(ollama-buddy-provider-create
 :name "LM Studio"
 :prefix "l:"
 :endpoint "http://localhost:1234/v1/chat/completions"
 :models-endpoint "http://localhost:1234/v1/models"
 :default-model "llama-3.2-3b-instruct")
#+end_src

There is also a dedicated module =ollama-buddy-openai-compat= that wraps this pattern with =defcustom= variables (see [[*Local OpenAI-Compatible Servers][Local OpenAI-Compatible Servers]]).

*** Provider create parameters

| Parameter          | Required | Description                                              |
|--------------------+----------+----------------------------------------------------------|
| =:name=            | yes      | Display name (e.g. "Groq")                               |
| =:prefix=          | yes      | Model prefix (e.g. "q:")                                 |
| =:endpoint=        | yes      | Chat completions API URL                                 |
| =:api-key=         | no       | String, function returning string, or nil                |
| =:api-type=        | no       | ='openai= (default), ='claude=, or ='gemini=             |
| =:models=          | no       | Static list of model names                               |
| =:models-endpoint= | no      | URL for dynamic model discovery                          |
| =:models-filter=   | no      | Predicate to filter discovered model IDs                 |
| =:default-model=   | no      | Default model name (without prefix)                      |
| =:temperature=     | no      | Float, default 0.7                                       |
| =:max-tokens=      | no      | Integer or nil                                           |
| =:extra-headers=   | no      | Additional HTTP headers alist                            |

** GitHub Copilot (=p:=)

GitHub Copilot is *not a traditional API*. It is a subscription service providing access to multiple LLM providers through GitHub's infrastructure. Ollama Buddy uses the chat API via OAuth authentication.

*** Available Models (via Copilot Chat)

*** Free Tier

- 50 premium requests/month at no cost

*** Notable Features

- Multi-model access through a single subscription
- Monthly cap on premium requests (not token-based)

*** How It Works

OpenRouter is an *aggregator/proxy* providing a single unified API to access 400+ models from all major providers. It passes through provider pricing with a 5.5% fee on credit purchases but no per-token markup.

*** Available Models

400+ models from OpenAI, Anthropic, Google, xAI, Mistral, DeepSeek, Meta (Llama), Cohere, and many others.

*** Pricing

- Pay-as-you-go, no minimums, no monthly fees
- You pay the underlying provider's price
- 5.5% fee when purchasing credits (only markup)

*** Free Tier

Dozens of models available for free:

- Without purchased credits: 50 requests/day
- With $10+ in purchased credits: 1,000 requests/day
- Free models: 20 RPM

*** Notable Features

- Single API for all providers -- ideal for testing
- Automatic fallback between providers
- Model comparison
- No commitment or lock-in
- OpenAI-compatible API format

*** Restrictions

- Free models have low rate limits
- Dependent on OpenRouter uptime in addition to underlying provider

* Local OpenAI-Compatible Servers

Any server that implements the OpenAI chat completions API can be used as a provider.  This gives you a second local inference path alongside Ollama -- useful for running models via different backends (GGUF via llama.cpp, GPTQ/AWQ via vLLM, etc.) or accessing models not available through Ollama.

** Quick setup with =ollama-buddy-provider-create=

The simplest approach -- one call in your init file:

#+begin_src elisp
(ollama-buddy-provider-create
 :name "LM Studio"
 :prefix "l:"
 :endpoint "http://localhost:1234/v1/chat/completions"
 :models-endpoint "http://localhost:1234/v1/models"
 :default-model "llama-3.2-3b-instruct")
#+end_src

No API key is needed for local servers.  Models are auto-discovered via the =/v1/models= endpoint.

** Using the dedicated compat module

If you prefer =defcustom= variables (for easier customisation via =M-x customize=):

#+begin_src elisp
(require 'ollama-buddy-openai-compat)
(setq ollama-buddy-openai-compat-base-url "http://localhost:1234")
(setq ollama-buddy-openai-compat-provider-name "LM Studio")
#+end_src

The module appends =/v1/chat/completions= and =/v1/models= to the base URL automatically.

Available customisation variables:

| Variable                                    | Default              | Description                          |
|---------------------------------------------+----------------------+--------------------------------------|
| =ollama-buddy-openai-compat-base-url=       | =http://localhost:1234= | Server base URL (no trailing slash)  |
| =ollama-buddy-openai-compat-provider-name=  | ="LocalAI"=          | Display name in status/headers       |
| =ollama-buddy-openai-compat-marker-prefix=  | ="l:"=               | Prefix for model names               |
| =ollama-buddy-openai-compat-api-key=        | =""=                 | Bearer token (empty for local)       |
| =ollama-buddy-openai-compat-models=         | =nil=                | Fallback model list                  |
| =ollama-buddy-openai-compat-default-model=  | =""=                 | Default model (empty = first found)  |
| =ollama-buddy-openai-compat-temperature=    | =0.7=                | Temperature (0.0-2.0)                |
| =ollama-buddy-openai-compat-max-tokens=     | =nil=                | Max tokens (nil = server default)    |
| =ollama-buddy-openai-compat-fetch-models=   | =t=                  | Auto-discover models on load         |

Use =M-x ollama-buddy-openai-compat-refresh-models= after changing the base URL at runtime.

** When to use local servers vs Ollama

Ollama is the default backend and is ideal for most local use cases.  Reasons you might add a second local server:

- *Different quantisation formats* -- llama.cpp/LM Studio for GGUF, vLLM for GPTQ/AWQ/FP16
- *Production serving* -- vLLM offers batching, continuous batching, and PagedAttention
- *Models not in Ollama's registry* -- load arbitrary HuggingFace models directly
- *GPU sharing* -- run Ollama on one GPU and vLLM on another
- *A/B testing* -- compare the same model across different inference engines

** Server offline handling

If the local server is not running when Emacs starts, model discovery will fail silently and fall back to the =:models= list.  Provide a static fallback list for servers that may not always be running:

#+begin_src elisp
(ollama-buddy-provider-create
 :name "LM Studio"
 :prefix "l:"
 :endpoint "http://localhost:1234/v1/chat/completions"
 :models-endpoint "http://localhost:1234/v1/models"
 :models '("llama-3.2-3b-instruct" "mistral-7b-instruct-v0.3")
 :default-model "llama-3.2-3b-instruct")
#+end_src

When the server comes online later, use =M-x ollama-buddy-provider-refresh= to re-discover models.
